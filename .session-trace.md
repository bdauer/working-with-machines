### Future: editorial review skill
The editorial guide's own register (terse, imperative, reference-doc) bleeds into reviewer agents' preferences. A reviewer prompt calibration line ("the target voice is conversational and exploratory, not the guide's reference-document register") fixes this per-invocation, but a proper skill would:
- Load the editorial guide as context
- Include the voice calibration instruction so reviewer agents don't mimic the guide's register
- Include a short voice sample from the author's actual writing for positive calibration (the guide describes the voice but doesn't demonstrate it)
- Set priority weighting: structural/voice issues > rhythm/tics > polish
- Handle doc-type branching automatically (detect which voice rules apply from frontmatter or file path)
